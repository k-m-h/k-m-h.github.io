Portfolio Website

September 2017 - December 2017

User Research

User Interviews

User interviews were coducted to provide a qualitative understanding of how users approach portfolios. The interviewees were asked questions about considerations they'd make if they were to make a portfolio of their own as well as expectations and concerns they'd have as users/viewers of a portfolio website.

Competitor Analysis

A Competitor Analysis was conducted to provide a qualitative understanding of other Software Engineer/Developer portfolios currently "on the market". The analysis provided valuable infromation about stylistic trends and the relative importance of different kinds of content.

Feature Value Matrix

A Feature Value Matrix was produced to quantify the relative value of various possible website feature by numerically ranking features in terms of severla different criteria. This matrix provided a strong basis for understanfing and comparing the value added by different possible features.

Feature Prioritization

The potential features of this portfolio website were visually organized based on the scores from the Feature Value Matrix in order to help inform the foundational building blocks and starting points for this portfolio.

User Testing

User Stories

User Stories were created to organize and brain storm how actual users would interact with the site.

Cognitive Walkthrough

In person cognitive walkthroughs were conducted to quickly identify pain points and usability flaws. Prospective users were given simple tasks to complete on the website, and given followup questions to help elucidate areas where users struggled or did not use the site as intended.

User Tests

Task based user testing was conducted via Usability Hub in order to quickly gather multiple reponses. Heatmaped click testsing with follow up questions helped to reveal discrepancies between user expectations and actual functionalities in areas such as email and linked in contact options.

Summary of Findings

This proccess of research and testing has provided the foundation for the usability of this site. Through inital User Research, I was able to prioritize the feaures of the portfolio into approcheable segments. Through competitor analysis, I was able to visualize the look and feel of the site that I was hoping to achieve as well how that type of design would translate to site components and content structuring. This made it easier to start sketching out the overall layout for the site. By keeping responsive design in the forefront, I was able to find layouts for the content that worked with varying screen sizes and user contexts.

Once the site had a working prototype, I was able to begin User Testing which exposed a whole new layer to my understanding of the user experience on this portfolio. For example, in my prototype, ym email and phone number were both clickable links. If a user clicke don the email address, it would launch the default email client for that device. My orginial concern with this feature would be that users would not trust this clickable link because they might not know what to expect from it and therefore not use it. My user testing however reavealed that most user were aware of what clicking the link would do but still prefered to manually copy and paste the address. The reason that was that most users did not actually use the default email client on their device and instead would have preferd to use the web client through their browser.

Incorporation of Findings

Thanks to user research and testing, I was able to come up with many solutions for improving the usbaility of my portfoilio website. One example of this is the solution I implemented for the email link issue described above in the Summary of Findings. I added a button for copying the email address to the clipboard and modifed the link text to clarify the action that would be caused by clicking it. I also did the same for the phone number.

Another area that was changed for user testing was the LinkedIn link. the prior design had a text that said find me in linked in, followed by a small LinkedIn icon. Javascript was used so that when clicked, the icon would open a small popup with a profile overview and a link to my profile. Cognitive walkthroughs revealed that some users would have expected a click on the text to take them to the profile, or to be taken directly once they clicked the icon. I addressed this issue by simply turning the item into a link that fit stylistically with the rest of the site.
